## 摘要
大语言模型（LLMs）在多种任务中表现出强大的性能，但其在长输入上下文下的推理受到内存容量和带宽的瓶颈限制。键值（KV）缓存的大小随序列长度线性增长，且在每个解码步骤中都需要从片外高带宽内存（HBM）重新读取到片上内存，导致推理受限于内存。现有方法通过剔除或量化来减少缓存，但通常将两者孤立对待。在本文中，我们将 KV 缓存压缩建模为一个率失真问题，在此框架下，剔除和量化是同一种比特分配方案的两个极端。这揭示了联合优化两者的必要性，从而促成了我们的方法 RDKV（率失真 KV 缓存压缩）。RDKV 根据压缩对注意力计算引起的失真来推导每个 token 或通道的权重。基于这些权重，它在预填充阶段后应用一次反向注水算法，为每个 token 或通道分配从全精度到零比特不等的位宽。在 LongBench、RULER 和 InfiniteBench 上的实验表明，RDKV 的平均性能优于最佳评估基线 9.1%。在 LongBench 上，它仅需 2.48% 的缓存保留率即可恢复 97.81% 的全缓存准确率。与全缓存 FlashAttention-2 解码相比，在 128K 上下文长度下，它实现了 4.5 倍的解码加速和 1.9 倍的峰值内存减少，同时保持了相当的性能。

## Abstract
Large language models (LLMs) have shown strong performance across diverse tasks, but their inference with long input contexts is bottlenecked by memory size and bandwidth. The Key-Value (KV) cache size grows linearly with sequence length and needs to be re-read from off-chip high-bandwidth memory (HBM) to on-chip memory at every decoding step, resulting in memory-bound inference. Existing methods reduce the cache by either eviction or quantization, but typically treat the two in isolation. In this paper, we cast KV cache compression as a rate-distortion problem, under which eviction and quantization are two end-points of the same bit allocation scheme. This exposes the need to optimize them jointly, motivating our method, RDKV (Rate-Distortion KV cache compression). RDKV derives the weight of each token or channel from the distortion that compression induces on the attention computation. Based on these weights, it assigns each token or channel a bit-width ranging from full precision down to zero bits guided by reverse water-filling, applied once after the prefilling stage. Experiments on LongBench, RULER, and InfiniteBench show that RDKV outperforms the best evaluated baseline by 9.1% on average. On LongBench it recovers 97.81% of full-cache accuracy with only 2.48% cache retention. Compared with full-cache FlashAttention-2 decoding, it achieves 4.5x decode speedup and 1.9x peak memory reduction with 128K context length, while maintaining comparable performance.

#### 1. 核心问题与研究动机
*   **核心问题**：大语言模型（LLMs）在处理长文本时，KV 缓存（Key-Value Cache）随序列长度线性增长，导致推理过程受限于内存容量和带宽（Memory-bound）。
*   **研究动机**：现有的 KV 缓存压缩技术主要分为**剔除（Eviction）**和**量化（Quantization）**两类。剔除方法（如 SnapKV）通过重要性评分丢弃不重要的 Token，而量化方法（如 KIVI）则降低所有 Token 的位宽。
*   **局限性**：现有方法通常将两者孤立处理。作者认为，剔除本质上是量化的一种极端形式（即分配 0 比特）。因此，需要一个统一的框架来联合优化剔除与量化，以在有限的比特预算下实现最小的精度损失。

#### 2. 方法论
RDKV 提出了一种基于**率失真（Rate-Distortion, R-D）理论**的压缩框架：
*   **核心思想**：将 KV 缓存压缩建模为一个受限优化问题——在给定的总比特预算下，通过为每个缓存单元（Token 或通道）分配最优位宽，使注意力计算的加权失真最小化。
*   **失真权重计算**：
    *   **V 缓存（Token 维度）**：根据 Token 在注意力矩阵中的累积权重（Attention Mass）定义权重 $w_t$。
    *   **K 缓存（通道维度）**：根据 Query 和 Key 矩阵对应列向量范数的乘积定义权重 $w_c$，以捕捉对 Logit 矩阵影响最大的异常通道。
*   **比特分配算法**：
    *   使用**逆向注水算法（Reverse Water-filling）**结合拉格朗日松弛法，在离散的硬件支持位宽集合 $\{0, 2, 4, 8, 16\}$ 中进行选择。
    *   权重高的单元分配高位宽（如 16-bit），权重极低的单元分配 0 比特（即被剔除）。
*   **TriZone 布局与硬件加速**：
    *   设计了 **TriZone** 存储布局，将混合位宽的缓存组织为三个区域（压缩区、全精度区、新生成区）。
    *   开发了定制化的 Triton 内核，将反量化过程融合进注意力计算中，避免了显存回写，从而实现真正的硬件加速。

*   **性能领先**：RDKV 在所有基准测试中均优于现有基线，平均领先幅度达 9.1%。

#### 7. 优点
*   **理论严谨**：首次将 KV 缓存压缩系统地建模为率失真问题，提供了坚实的数学基础。
*   **联合优化**：打破了剔除与量化的界限，通过单一目标函数动态决定每个单元的去留与精度。
*   **软硬件协同**：不仅有算法创新，还通过 TriZone 布局和融合内核解决了混合位宽在 GPU 上的执行效率问题。

*   **静态分配限制**：目前的比特分配是在 Prefill 阶段后一次性完成的，无法在长文本生成过程中根据注意力模式的动态漂移进行实时重新分配。
*   **校准依赖**：算法依赖于预先计算的经验失真表（Calibration），虽然模型间差异较小，但仍需针对特定架构进行初步校准。
*   **应用场景**：主要针对“长输入、短输出”的场景优化明显，若生成内容极长，其静态分配策略的有效性可能下降。
