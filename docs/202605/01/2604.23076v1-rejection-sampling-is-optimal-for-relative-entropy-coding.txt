Abstract —In relative entropy coding, a sender aims to design a stochastic code such that, on input X ∼ PX , the receiver can generate a sample Y ∼ PY |X . It is a standard result that (1) this requires at least I(X; Y ) bits, (2) the lower bound is achievable within a logarithmic gap, and (3) this gap cannot be reduced in general. The necessity of the gap suggests that the mutual information is not the correct information measure to quantify the rate of relative entropy coding. A potential alternative emerged in the work of Flamich et al. [1], who proved a tighter lower bound of IF(X → Y ), a quantity we call the functional information . In this paper, we show this lower bound is tight by constructing the ring toss code ,an encoding method for rejection sampling which uses at most 

IF(X → Y ) + log e bits. This demonstrates that rejection sampling is optimal for relative entropy coding. Our result implies that the classical mutual information lower bound is achievable within log( I(X; Y ) + 1) + 2 .45 bits in general and within 1.45 bits for singular channels, which are both the tightest bounds of their kind to date. Moreover, our one-shot result also recovers Sriramu and Wagner’s asymptotic results on the second-order redundancy of relative entropy codes. 

Index Terms —Relative entropy coding, channel simulation, rejection sampling, ring toss code, singular channels. 

I. I NTRODUCTION 

For a pair of random variables X, Y ∼ PX,Y , the goal of relative entropy coding (also called channel simulation) is to find a stochastic code [2] that, given an input X ∼ PX ,encodes a single sample Y ∼ PY |X in finitely many bits. In doing so, the sender and receiver simulate the channel X → Y .Formally, a stochastic code is a triplet (Z, enc , dec) where  

> •

Z is a random variable independent of X shared by encoder and decoder called the common randomness ; 

> •

enc : X × Z → { 0, 1}* is the encoder mapping an input and common randomness realization to a binary string;  

> •

dec : {0, 1}* × Z → Y is the decoder mapping the en-coder output and common randomness to a point y ∈ Y .The formal requirement of (exact) relative entropy coding is that for all inputs x ∈ X ,

dec(enc( x, Z ), Z ) ∼ PY |X=x. (1)

Under this requirement, it can be shown that [3], [4] 

I(X; Y ) ≤ H(Y | Z) ≤ E[|enc( X, Z )|], (2) where | · | is the length of a binary string. There is a long line of research work [3]–[6] to design stochastic codes which almost achieve this lower bound, the crowning achievement of which is the Poisson functional representation [4], yielding 

H(Y | Z) ≤ I(X; Y ) + log( I(X; Y ) + 1) + 4 . (3) This logarithmic redundancy over the mutual information is also achieved by other schemes, including greedy rejection sampling [3], [6] and greedy Poisson rejection sampling [7]. However, it is not possible to remove this log term, as there exist channels for which I(X; Y ) + log( I(X; Y ) + 1) − 1

bits are necessary [4], [5]. Thus, a natural question is whether a different information measure and encoding scheme could yield a tight bound. In [1], progress was made in this direction by improving the lower bound on any relative entropy code to 

IF(X → Y ) ≤ H(Y | Z), (4) where IF(X → Y ) is the functional information (see Defini-tions 2 and 3). Note that I(X; Y ) ≤ IF(X → Y ), so (4) is at least as tight as (2). However, the lower bound in (4) has only been shown to be achievable (within log e bits) for additive exchangeable noise channels with finite field input [8]. In this paper, we construct the ring toss code , an encoding method for standard rejection sampling. We prove for general channels that it achieves the lower bound in (4) within an additive constant of log e, demonstrating that rejection sampling is optimal for relative entropy coding .Concretely, our contributions are as follows:  

> •

We construct the ring toss code and prove that its one-shot rate satisfies H(Y | Z) ≤ IF(X → Y ) + log e. This upper bound matches the lower bound in (4) within an additive constant of log e ≈ 1.44 . The algorithm and proof are remarkably simple, relying entirely on a changed encoding of the index returned by standard rejection sampling.  

> •

For a class of channels called singular channels [9], [10] (see Definition 4), we prove that IF(X → Y ) = I(X; Y ).Therefore, our work identifies a large class of channels 

> arXiv:2604.23076v1 [cs.IT] 25 Apr 2026

for which we can achieve constant non-asymptotic redun-dancy over the mutual information.  

> •

For general channels, our result implies the tighter achiev-ability result in terms of the mutual information 

H(Y | Z) < I (X; Y ) + log( I(X; Y ) + 1) + 2 .45 . (5)  

> •

Finally, our one-shot result recovers the asymptotic char-acterization of relative entropy coding in [1], [10], and is achieved by a simpler scheme with optimal redundancy. 

A. Notation 

In this paper, log denotes the binary logarithm, and all information measures are in bits. For an event A, its indicator function is 1[A]. Unless otherwise stated, all random variables are assumed to take values in some arbitrary Polish space. Random variables are written in capital letters X, we write 

X ∼ PX to say that X has probability measure PX , and use P ⊗nX for n independent copies of PX . Random variable alphabets or spaces are denoted by caligraphic letters X ∈ X ,

Y ∈ Y . We denote the expectation of X and the condtitional expectation of X given Y as E[X] and E[X | Y ], respectively. We briefly describe the rejection sampling algorithm, which forms the basis of the ring toss code; see [2], [13] for a detailed treatment. We have defined the ring toss code , an encoding method for standard rejection sampling, which we prove is optimal for relative entropy coding. In doing so, we provide a matching achievability bound to the one-shot lower bound of Theorem III.1 in [1]. For singular channels, our upper bound achieves a constant redundancy over the mutual information. The ring toss code also provides a provably tight strong functional rep-resentation lemma [4], which has implications across a variety of problems in lossy compression and machine learning. There are several interesting directions of future work stemming from our results. Most immediately, it is of interest to remove the boundedness condition dP Y |X  

> dP Y∞

≤ M .We emphasize that this is a requirement of the rejection sampling algorithm, and are optimistic that the insight of encoding the probability of hitting a set in X -space can be adapted to other relative entropy codes. Moreover, [12] shows that any sampling algorithm will have expected run-time E[K] ≥ dP Y |X 

> dP Y∞

. Thus, the boundedness assumption is equivalent to the algorithm terminating almost surely. The functional information also has several interesting open questions about its properties. For instance, are there classes other than singular channels for which the gap between
