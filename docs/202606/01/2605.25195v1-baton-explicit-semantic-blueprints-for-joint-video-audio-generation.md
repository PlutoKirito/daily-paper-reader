## 摘要
当前的开源扩散模型在生成稳定且同步的视听内容时面临困难，尤其是在需要复杂语义推理的场景中。根本原因在于现有方法依赖现成编码器提供的粗粒度文本嵌入来引导音频-视频去噪，这丢弃了细粒度语义，并且关键的是，缺乏共享的长程规划，导致去噪轨迹不协调以及跨模态对齐脆弱。我们提出Baton，这是首个将显式语义规划引入联合视频-音频生成的框架。我们的关键洞察是，用语义丰富、模态感知的规划token来补充粗粒度文本引导，并在去噪前进行联合推理和相互对齐，可以同时恢复细粒度语义细节，并建立协调音频和视频去噪轨迹的共享蓝图。具体而言，Baton首先引入VA-Planner，这是一个配备双语义对齐塔的多模态语言模型，其中可学习查询与视频和音频特征进行交叉注意力，生成一对语义对齐的视频和音频规划token作为关键帧级别的蓝图。这些规划token通过交叉注意力层注入扩散骨干网络，提供与粗粒度文本嵌入互补的时间引导。由于规划token与扩散潜变量之间不存在一一对应的时空对应关系，我们进一步提出相对语义RoPE，这是一种相对位置编码，将规划token和潜变量映射到共享的时空坐标框架中，使每个潜变量能够准确关注其位置对应的语义线索。基准实验结果表明，Baton在定性和定量上均有效。

## Abstract
Current open-source diffusion models struggle to generate stable and synchronized audio-visual content, particularly in scenarios demanding complex semantic reasoning. The root cause is that existing methods rely on coarse text embeddings from off-the-shelf encoders to guide audio-video denoising, which discards fine-grained semantics and, critically, lacks a shared long-horizon plan, leading to uncoordinated denoising trajectories and fragile cross-modal alignment. We propose Baton, the first framework that introduces explicit semantic planning into joint video-audio generation. Our key insight is that complementing coarse text guidance with semantically rich, modality-aware planned tokens, jointly reasoned and mutually aligned before denoising, can simultaneously restore fine-grained semantic detail and establish a shared blueprint that coordinates both audio and video denoising trajectories. Concretely, Baton first introduces the VA-Planner, a multimodal language model equipped with dual semantic alignment towers, where learnable queries cross-attend to both video and audio features to produce a pair of semantically aligned video and audio planned tokens as keyframe-level blueprints. These planned tokens are injected into the diffusion backbone via cross-attention layers, providing temporally grounded guidance complementary to coarse text embeddings. Since planned tokens do not share one-to-one spatial-temporal correspondence with diffusion latents, we further propose Relative Semantic RoPE, a relative positional encoding that maps planned tokens and latents into a shared spatial-temporal coordinate frame, enabling each latent to accurately attend to its positionally corresponding semantic cues. Experiments on benchmarks show the effectiveness of Baton both qualitatively and quantitatively.

## 1. 核心问题与整体含义（研究动机和背景）

现有开源扩散模型在生成稳定且同步的视听内容时表现不佳，尤其在需要复杂语义推理的场景（如涉及多阶段动作、人-物交互、多说话人对话等）中问题突出。根本原因在于：现有方法依赖现成文本编码器输出的粗粒度全局文本嵌入来引导音频和视频的去噪过程。这种嵌入丢弃了细粒度语义信息，更重要的是缺乏一个共享的、长程的时序规划，导致视频和音频两条去噪轨迹相互独立且不协调，容易产生对抗性偏离，造成跨模态对齐脆弱甚至失真。尽管有些工作（如LTX-2、MOVA）利用LLM增强提示细节，但它们仍是全局嵌入注入扩散模型，无法从根本上解决问题。Baton首次提出在联合视频-音频生成中显式地分离语义规划与生成，建立一个模态感知的共享蓝图，协调两条去噪轨迹。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 在去噪之前，先通过显式的语义规划生成一对模态对齐的、富含语义的“规划 token”（planned tokens）作为关键帧级别的蓝图，并补全粗粒度文本引导。这个蓝图同时包含视频和音频的细粒度语义结构，并预先建立了跨模态一致性。
- 规划 token 通过交叉注意力注入扩散骨干网络（DiT），提供时序锚定的语义引导。
- 由于规划 token 与扩散潜变量（latents）之间存在时空网格不匹配，引入相对语义 RoPE（RS-RoPE）将两者映射到共享的坐标框架，使每个潜变量能准确关联到位置对应的语义线索。

### 关键技术细节

#### VA-Planner（视频-音频规划器）
- 采用可训练的多模态大语言模型（MLLM，初始化自 Qwen3-8B），接收用户提示，生成结构化输入：系统提示 + 视频提示 + 音频提示 + 特殊分隔token（`<|v_start|>`... 等占位符）。
- 通过自回归推理，从 MLLM 的隐藏状态中提取视频隐藏状态 Hv 和音频隐藏状态 Ha。
- 引入双语义对齐塔（dual semantic alignment towers），每个塔使用可学习查询（learnable queries）对 Hv 和 Ha 进行交叉注意力，并执行跨模态注意力（CMAttn）实现双向信息交换，使视频蓝图也能感知音频信息、音频蓝图也能感知视频信息。
- 最后通过 Sem-MLP 将输出投影到目标感知编码器（SigLip2 用于视频，WavTokenizer 用于音频）的特征空间，得到最终的规划 token：Hsem_v 和 Hsem_a。
- 训练时使用 L2 损失回归连续特征（来自冻结的 SigLip2 和 WavTokenizer 的中间层特征），而非离散 token。

#### Relative Semantic RoPE（RS-RoPE）
- 针对规划 token 与扩散潜变量之间时空网格不一致的问题（视频潜变量在 (T_lv, H_lv, W_lv) 网格上，规划视频 token 在 (N, hs, ws) 网格上；音频潜变量在 T_la 维上，规划音频 token 在 La 维上）。
- 对视频交叉注意力（VCAttn）：将查询位置索引和键位置索引通过比例因子映射到共享坐标，然后应用 3D RoPE。
- 对音频交叉注意力（ACAttn）：使用 1D RoPE，将音频潜变量与规划音频 token 对齐到同一时间轴。
- 这种相对位置编码使得每个潜变量能够关注时空上对应的规划语义，而不是将结构化蓝图压平成全局信号。

#### 训练流程（三阶段）
1. **Stage 1 (VA-Planner 预训练)**：训练整个 VA-Planner（MLLM + Sem-MLP），以冻结的 SigLip2 和 WavTokenizer 特征作为回归目标，学习预测规划 token。
2. **Stage 2 (DiT 适配)**：初始化 DiT（来自 Ovi），将真实地标特征 Fgt_v、Fgt_a 直接注入交叉注意力层，训练 DiT 和 Latent-MLP，采用流匹配损失（flow matching loss）。
3. **Stage 3 (联合微调)**：冻结 VA-Planner，DiT 继续训练，此时接收 VA-Planner 预测的规划 token 作为条件，以弥合 Stage 2 中理想特征与真实预测之间的差距，减少曝光偏差。

## 6. 主要结论与发现
1. Baton 在 Verse-Bench（简单场景）上取得与 SOTA 相当的结果，但在 Sem100（复杂语义场景）上显著超越所有开源方法：P-Acc 提升 32%，M-WER 降低 76%，DeSync 改善 30%。这表明显式语义规划对于复杂提示至关重要。
2. VA-Planner 生成的规划 token 是与粗粒度文本互补的核心语义驱动力；仅使用规划 token 已能取得强结果，全局文本嵌入提供整体先验。
3. 双语义对齐塔（特别是跨模态注意力）和塔内 RoPE 是关键：去除塔或消除 RoPE 会导致严重的时序错位和性能下降。
4. RS-RoPE 对解决规划 token 与潜变量之间的时空不匹配至关重要，无 RS-RoPE 时性能甚至低于无 VA-Planner 基线。
5. 三阶段训练课程（先训练规划器、再用真实特征适配 DiT、最后微调适应预测）是必要的，跳过 Stage-2 会导致性能大幅下降。
6. 更大规模的 MLLM 骨干（32B）带来更好性能，但计算成本高；Qwen3-8B 在性能和效率间取得良好平衡。
7. Baton 在提示遵循能力上可与商业模型（Veo 3.1, Seedance 2.0 等）相媲美，甚至在某些复杂场景下更优，但在视频保真度和音频美感上仍有差距。
8. 用户研究结果强烈支持 Baton 优于所有对比方法。

## 7. 优点
- **创新性**：首次在联合视频-音频生成中引入显式语义规划，将语义推理与生成分离，解决了粗粒度引导和轨迹不协调的根本问题。
- **技术设计**：VA-Planner 的双塔结构、跨模态注意力、回归连续特征（而非离散 token）、RS-RoPE 位置映射等设计精巧合理。
- **实验充分**：大规模消融实验覆盖了方法各个细节，验证了每个组件的必要性。
- **性能显著**：在复杂场景下各项指标大幅超越现有开源方法，尤其是 M-WER 和 DeSync 提升巨大，表明其处理多说话人、长程事件的能力突出。
- **实用性**：与商业模型的比较显示其在语义理解上的竞争力，具有实际应用潜力。
- **开源友好**：基于 Qwen3-8B 和 Ovi 等开源组件构建，方法易于复现和扩展。

## 8. 不足与局限
- **计算开销**：引入了额外的 VA-Planner 推理（MLLM 自回归生成），相比基础 DiT（Ovi）增加了约 28% 的显存和 76% 的推理时间（346s vs 184s）。尽管权衡有利，但在资源受限场景下可能受限。
- **视频细节保真度**：在涉及多人物且每个人物在画面中尺寸较小时，合成的人脸模糊、缺乏细节。作者归因于关键帧空间 token 数量有限以及 3D VAE 压缩。未来可结合人脸增强或自适应 token 分配。
- **未见完整的算力说明**：未报告训练所需 GPU 数量和总时长，影响可复现性评估。
- **测试偏差**：Sem100 为自建数据集，虽然消融实验在同一数据集上公平，但泛化性可能需要更多独立第三方基准验证。
- **评价指标局限**：P-Acc 使用 Gemini 评估，可能引入黑箱模型偏差；M-WER 依赖 Qwen3-Omni 进行说话人分离和转录，错误会累积。
- **伦理风险**：可能被用于制作深度伪造视频、虚假信息传播。作者提出需集成水印、内容检测、使用约束等措施，但未实际实现。
- **当前模型为“冻结”VA-Planner 的联合微调**：作者将 VA-Planner 冻结后训练 DiT，可能限制了规划器与生成器的协同优化潜力。未来工作或可考虑端到端训练。
