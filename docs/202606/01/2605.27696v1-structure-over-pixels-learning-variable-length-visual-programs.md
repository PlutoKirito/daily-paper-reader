## 摘要
离散视觉分词器将图像转换为有序的代码序列，为场景的结构化描述提供了自然表示。然而，现有的自适应分词器要么需要事后搜索，要么在预训练的离散速率集合中进行选择，而不是学习与模型和场景耦合的连续每图像序列长度，并且它们通常针对像素重建进行训练，强调纹理而非结构。我们提出STROP，一种离散视觉分词器架构，形成结构化场景表示，并同时学习图像的视觉程序应有多长。使用由冻结DINOv3特征的局部率失真探针监督的四阶段课程，STROP优化了一个专门的长度头部，在单次前向传递中估计活动前缀长度。通过绕过像素级重建梯度，码本完全由更高级潜在表示的质量塑造。程序长度随场景复杂度增长，组合结构的迹象出现在下游密集预测迁移以及直接检查学习到的代码词汇中。

## Abstract
Discrete visual tokenizers translate images into ordered sequences of codes, providing a natural representation for structural description of scenes. Yet existing adaptive tokenizers either require post-hoc search or select among a discrete set of pre-trained rates, rather than learning a continuous per-image sequence length coupled to the model and scene, and they typically train against pixel reconstruction, emphasizing texture rather than structure. We propose STROP, a discrete visual tokenizer architecture that forms structural scene representations and simultaneously learns how long an image's visual program should be. Using a four-phase curriculum supervised by local rate--distortion probes against frozen DINOv3 features, STROP optimizes a dedicated length head that estimates the active prefix length in a single forward pass. By bypassing pixel-level reconstruction gradients, the codebook is shaped entirely by the quality of higher-level latent representations. Program length grows with scene complexity, and signs of compositional structure emerge both in downstream dense-prediction transfer and in direct inspection of the learned code vocabulary.

- **问题：** 现有的离散视觉分词器（DVT）如VQ-VAE、VQGAN，通常将图像映射到固定数量的离散代码，序列长度在图像可见前就已确定，无法适应场景复杂度（例如平滑天空和拥挤街道使用相同码本预算）。少数自适应方法要么需要事后搜索（如通过阈值选择前缀），要么在少量预训练速率中选择，未能学习连续的、与图像内容和模型耦合的序列长度。此外，这些方法普遍依赖像素级重建损失，从而强调纹理细节而非结构化语义，不利于形成可解释、可操纵的场景表示。

- **动机：** 受认知原则（格式塔、最小描述长度、奥卡姆剃刀）启发，视觉表征的代码数量应与场景中的“视觉结构量”相关。同时，特征蒸馏（如DINO、BEiT v2）表明以教师特征为训练信号可以产生更语义化的潜在空间。结合这两点，本文旨在学习一种能够自动根据图像内容决定编码长度的离散视觉程序，且训练过程避开像素重建，直接由高层次特征质量驱动码本学习。

- **整体含义：** 提出STROP，一种生成-解释器架构（generator-interpreter），能够为每张图像生成变长的离散代码序列（称为“程序”），该序列经解释后可重建对齐DINOv3的补丁特征场。程序长度与场景复杂度正相关，单个代码表现出空间连贯的语义句柄，展现出组合结构。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 使用冻结的DINOv3 ViT编码器提取补丁特征，通过一个编码器-仅变压器生成器将补丁特征映射到有序的离散代码序列。
- 通过一个可学习的长度头（MLP）从编码器特征直接预测每张图像应使用的代码长度（前缀长度），实现单次前向传播即可完成变长截断。
- 训练目标主要来自与冻结DINO特征的潜在对齐损失（余弦相似度+MSE），不使用像素级重建梯度（卷积解码器使用stop-gradient，仅作为检查表面）。
- 采用四阶段课程学习来训练长度头，使其能从率-失真探针中学习最优截断点。

### 关键技术细节
- **架构五阶段：**
  1. **编码器：** 冻结的DINOv3 ViT，输出补丁特征序列（S个token）。
  2. **程序生成器：** 编码器-仅transformer，将补丁特征与K个可学习查询token拼接，使用因果掩码确保查询按序生成；输出K个预量化token。
  3. **向量量化：** 码本大小|CB|=1024，维度dc=16，ℓ2归一化，EMA更新。
  4. **长度头：** sigmoid MLP，对编码器特征均值池化后输出ˆL = K·σ(h(pool(Ze)))，训练好后在推理时直接使用。
  5. **解释器：** 镜像生成器，使用网格查询token交叉注意力到被选中的前缀代码，恢复为空间特征图 ˆF。
- **训练损失：** `L = Llat + Lcommit + Ldiv + Llen`
  - `Llat`：余弦+MSE vs 冻结DINO特征。
  - `Lcommit`：标准VQ承诺损失。
  - `Ldiv`：多样性损失（防止码本坍塌）。
  - `Llen`：在阶段3激活，MSE损失监督长度头预测与oracle目标。
- **四阶段课程（详见附录A）：**
  - 阶段1：随机Beta截断（偏向长前缀），使信息前置到早期token。
  - 阶段2：oracle目标估计：基于重建错误和局部率-失真曲线斜率计算每个样本的理想长度，仅累积不用于截断。
  - 阶段3：激活长度头，监督学习oracle目标，截断仍随机。
  - 阶段4：逐步从随机截断过渡到长度头预测截断（线性ramp），最终全部用预测长度。
- **码本设计：** 1024个16维代码，ℓ2归一化，EMA更新τ=0.95，与低代码维度结合防止死码漂移。

### 算法流程（文字说明）
1. 输入图像→冻结DINO编码器得到补丁特征Ze。
2. 生成器：Ze与K个查询token拼接，经因果掩码 transformer，输出K个预量化token。
3. 长度头：从Ze均值池化后经MLP预测长度ˆL（在推理时使用）。
4. 向量量化：每个预量化token映射到最近码本索引，获得量化token Zq。
5. 截断：保留前ˆL个量化token，其余掩码。
6. 解释器：网格查询交叉注意力到保留token，输出特征图 ˆF。
7. 损失计算：与冻结DINO特征对比，更新生成器、码本等（解码器stop-gradient）。

## 7. 优点

- **方法创新性：** 四阶段课程学习可变的程序长度，无需事后搜索或离散速率选择；长度头单次前向预测，高效。
- **训练信号创新：** 完全基于冻结DINO特征的潜在对齐，避免像素重建带来的纹理偏差，强调结构化语义。
- **架构简洁有效：** 生成器-解释器均采用编码器-仅变压器，因果掩码保证顺序；解释器使用交叉注意力恢复空间场，实现位姿的可压缩编码。
- **实验分析深度：** 不仅评估下游任务，还深入分析程序结构（擦除归因、协同效应、码本统计、LLM解码），多角度验证结构涌现。
- **代码和结果开源：** 基于公开的DINOv3，实验设置详细，便于复现。

## 8. 不足与局限

- **原始程序可读性低：** 直接在量化token上训练的线性探针远弱于通过解释器后的表示（gap大），因此程序应被视为可执行媒介而非直接可读特征，token的顺序和交互尚未被探针充分捕获。
- **潜在对齐不完美反映迁移：** DINO-S编码器重建分最高，但DINO-B编码器在下游任务中更优（即使重建分较低），说明当前对齐度量（余弦/均方误差）未完全捕捉对下游有用的方向性结构。
- **Cityscapes性能落后FlexTok：** 在密集街景场景中，FlexTok凭借更大的训练信号和像素感知解码取得优势，STROP受限于COCO训练分布和DINO教师的选择，可能是未来改进点。
- **LLM属性绑定失败：** 学习到的码本未能可靠地将对象属性（颜色、材质）编码为可被LLM解码的符号绑定，限制了作为透明符号接口的用途。
- **实验覆盖有限：** 未与更多固定长度tokenizer（如ViT-VQGAN）进行下游对比，未研究更大码本（>1024）或更长最大长度（>64）的影响，仅在CLEVR上做了码本大小消融。
- **资源报告略简：** 只报告了单卡训练时长，未提供总GPU小时或扩展性分析。
- **课程复杂度：** 四阶段课程超参数（β, δ, τ, 调制因子）需精细调整，可能对可复现性和鲁棒性有影响。
