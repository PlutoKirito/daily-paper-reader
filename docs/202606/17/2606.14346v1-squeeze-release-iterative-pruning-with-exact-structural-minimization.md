## 摘要
非结构化剪枝产生稀疏权重张量，但标准实现保持张量形状不变，因此部署的模型并不比剪枝前更小。我们提出了一种精确的结构重写，称为最小化，它将掩码网络转换为一个更小的密集网络，其前向函数与原始网络在浮点舍入误差范围内一致。挤压-释放循环迭代进行剪枝和最小化，中间包含一个释放步骤，该步骤将压缩张量内的精确零位置重新启用为小的校准噪声，从而将原本浪费的容量转化为可训练参数。连续的循环利用该容量来发现单次剪枝无法达到的结构冗余。我们还引入了CompensatedLayerNorm，这是一种保持函数不变的LayerNorm替代方案，它将最小化扩展到跨LayerNorm残差流的通道缩减。在一个全连接模型网络上，挤压-释放将可部署网络压缩至未剪枝模型的39倍，在现代CNN（ConvNeXt-Tiny）上压缩至14.8倍，同时保持相当的精度。此外，我们证明该重写可以扩展到Transformer架构。

## Abstract
Unstructured pruning produces sparse weight tensors, but the standard implementation keeps tensor shapes unchanged so the deployed model is no smaller than before pruning. We present an exact structural rewrite, which we call minimization, that converts a masked network into a smaller dense network with the same forward function up to floating-point rounding. The Squeeze-Release cycle iterates pruning and minimization with an intermediate release step that re-enables the exact-zero positions inside the compacted tensors as small calibrated noise, turning otherwise wasted capacity back into trainable parameters. Successive cycles use that capacity to find structural redundancy a single pass cannot reach. We additionally introduce CompensatedLayerNorm, a function-preserving replacement for LayerNorm that extends minimization to channel reduction across LayerNorm-equipped residual streams. Squeeze-Release compresses the deployable network to 39x smaller than the unpruned model on a fully-connected model network and 14.8x smaller on modern CNN (ConvNeXt-Tiny), at comparable accuracy. In addition we prove that the rewrite can be extended to transformer architectures.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：非结构化剪枝（unstructured pruning）虽然将大量权重置为零，但标准实现（如 PyTorch 的 `torch.nn.utils.prune`）保持张量形状不变，部署模型的实际内存和计算开销并未减少。常用的“mask-alive”参数计数（即非零权重数量）并不能反映可部署模型的大小，因为零权重仍被存储和参与计算，且通用 GPU 对任意稀疏模式加速效果有限。
- **整体含义**：本文旨在解决“如何将剪枝后稀疏网络真正压缩为更小的稠密网络”这一结构性重写问题。提出**精确最小化（minimization）**，将掩码网络转换为前向函数等价的更小稠密网络；进一步提出**Squeeze-Release 迭代循环**，通过释放步骤将压缩张量中的零位重新激活为可训练参数，多次循环挖掘单次剪枝无法达到的结构冗余；并引入**CompensatedLayerNorm** 支持跨 LayerNorm 残差流的精确通道缩减，将方法扩展至现代 Transformer 架构。

## 2. 方法论

### 核心思想
- 精确最小化：检测并移除**死输入**（行全零）和**死输出**（列或滤波器全零）单元，通过偏置折叠等操作保持前向函数精确等价。
- Squeeze-Release 循环：交替执行 Prune → Squeeze（最小化）→ Release → Fine-tune。
- CompensatedLayerNorm：存储移除通道的统计量（数量 K、和 S、平方和 Q），在缩减后的输入上重建全宽 LayerNorm 输出。

### 关键技术细节
- **Prune**：使用梯度×权重大小作为重要性分数，对 FC 层逐标量剪枝，对 Conv2d 按滤波器粒度剪枝（平均值聚合），全局统一排序。
- **Squeeze（最小化）**：
  - 对 FC：检测行/列全零，死输入将常数输出折叠到下一层偏置并移除对应行/列；死输出直接移除。
  - 对 ConvNeXt 块：分三种模式：
    - **Mode A**：缩减反转瓶颈内部 4C 维度（PWConv1→GELU→PWConv2）。
    - **Mode B**：缩减残差流通道宽度（DWConv→LayerNorm→PWConv1），使用 CompensatedLayerNorm。
    - **Mode C**：移除整个块（所有 DWConv 滤波器全零时），将常数输出向后折叠。
- **Release**：将压缩张量中精确为零的权重替换为小高斯噪声（均值 0，标准差来自存活权重的 0.01 倍），恢复可训练容量，不改变张量形状。
- **CompensatedLayerNorm**：保存移除通道的 K、S、Q，在每次前向时根据存活通道的均值和二阶矩重建全局均值和方差，保证输出与原全宽 LayerNorm 一致。公式：μ = (m μ_A + S)/(m+K)，σ² 类似复合计算。
- **循环终止条件**：验证准确率低于阈值（80%）或单次下降超过 10 个百分点，或达到最大循环次数（ConvNeXt 限制 100）。

## 6. 主要结论与发现

1. **精确最小化是有效的压缩后处理**：单次 post-hoc 最小化即可将 No-minimize 基线的可部署参数从 191k 降至 18k（FC，10.5×），从 27.8M 降至 10.0M（ConvNeXt，2.78×）；ViT 降至 19.4M（12% 缩减）。
2. **Squeeze-Release 进一步显著压缩**：FC 可部署参数降至 4,878（39.2× 对原模型），ConvNeXt Mode AB 降至 1.88M（14.8× 对原模型），精度与基线无统计显著差异（FC 97.03% vs 95.99%，ConvNeXt 90.27% vs 90.81%）。
3. **mask-alive 与可部署大小的鸿沟**：No-minimize 基线中可部署大小是 mask-alive 计数的 5.4×（FC）、11.3×（ConvNeXt）、5.0×（ViT），表明 mask-alive 指标严重高估实际压缩效果。
4. **释放步骤的关键作用**：Squeeze-Release 循环次数远多于 No-minimize（ConvNeXt 平均 97.2 次 vs 3.6 次），说明释放步骤将浪费容量转化为可训练参数，使网络持续发现新冗余。
5. **CompensatedLayerNorm 使残差流缩减可行**：Mode AB 比 Mode A 进一步压缩 1.3×（2.45M → 1.88M），精度略有下降但可接受，验证了其有效性。
6. **可扩展至 Transformer**：ViT 的后验最小化证明最小化重写可应用于注意力机制和 MLP 块，没有架构限制。

## 7. 优点

- **方法创新性强**：将精确结构重写与迭代释放结合，解决了非结构化剪枝的“部署无效”问题。CompensatedLayerNorm 是首个精确保持 LayerNorm 输出的通道缩减方案。
- **函数保持变换**：最小化在前向函数完全等价（浮点误差 10⁻⁶ 级）的前提下缩小网络，无需重新训练即可部署。
- **指标公平性**：同时报告 mask-alive 和可部署参数，揭示常被忽视的鸿沟，推动更诚实的压缩报告。
- **实验设计严谨**：多随机种子、统计检验、消融实验、不同架构验证。释放步骤的初始化分布经消融（高斯 vs 直接置零）确认效果类似，核心是“恢复可训练容量”。
- **实用性强**：Mode A 几乎零额外部署成本即可显著压缩；CompensatedLayerNorm 为轻量级 drop-in 替换，便于集成。

## 8. 不足与局限

- **计算成本高**：Squeeze-Release 的循环次数远多于基线，特别是在 ConvNeXt 上限制 100 循环后仍有提升空间，实际训练时间增加 >30 倍。对于一次性压缩场景，成本仍可接受，但并非免费。
- **ViT 实验不完整**：仅做了单种子的 post-hoc 最小化，未跑完整 Squeeze-Release 循环，缺乏对 Transformer 迭代效果的验证。
- **未涉及大语言模型**：虽然理论上可扩展，但未在 GPT/LLaMA 等大规模模型上实验，实际部署中的内存带宽、顺序依赖等问题未处理。
- **释放步骤初始化敏感度**：虽然“分布选择关键性较低”，但噪声尺度（0.01 倍存活统计）需手动调节，不同架构可能需要微调。
- **终止条件可能过早**：80% 验证准确率阈值是启发式设定，对于更困难的任务（如 ImageNet）可能需要更低阈值或更精细策略，影响最终压缩比。
- **仅考虑权重剪枝**：偏置、归一化仿射参数、嵌入层未被剪枝，限制了最大压缩比。
- **对比基线有限**：未与最新结构化剪枝（如 SliceGPT、Minitron）直接比较，仅与标准非结构化迭代剪枝加后处理对比。
