## 摘要
非结构化剪枝产生稀疏的权重张量，但标准实现保持张量形状不变，因此部署的模型并不比剪枝前的模型更小。我们提出一种精确的结构重写，称为最小化，它将掩码网络转换为更小的密集网络，其前向函数在浮点舍入误差范围内相同。挤压-释放循环迭代进行剪枝和最小化，并加入一个中间释放步骤，该步骤将压缩张量中的精确零位置重新启用为小的校准噪声，从而将原本浪费的容量转化为可训练的参数。连续的循环利用这种容量来找到单次剪枝无法触及的结构冗余。我们还引入了补偿层归一化（CompensatedLayerNorm），一种保持功能的层归一化替换，将最小化扩展到具有层归一化的残差流中的通道缩减。在全连接模型网络上，挤压-释放将可部署网络压缩到比未剪枝模型小39倍，在现代CNN（ConvNeXt-Tiny）上小14.8倍，同时保持相当的精度。此外，我们证明了该重写可以扩展到Transformer架构。

## Abstract
Unstructured pruning produces sparse weight tensors, but the standard implementation keeps tensor shapes unchanged so the deployed model is no smaller than before pruning. We present an exact structural rewrite, which we call minimization, that converts a masked network into a smaller dense network with the same forward function up to floating-point rounding. The Squeeze-Release cycle iterates pruning and minimization with an intermediate release step that re-enables the exact-zero positions inside the compacted tensors as small calibrated noise, turning otherwise wasted capacity back into trainable parameters. Successive cycles use that capacity to find structural redundancy a single pass cannot reach. We additionally introduce CompensatedLayerNorm, a function-preserving replacement for LayerNorm that extends minimization to channel reduction across LayerNorm-equipped residual streams. Squeeze-Release compresses the deployable network to 39x smaller than the unpruned model on a fully-connected model network and 14.8x smaller on modern CNN (ConvNeXt-Tiny), at comparable accuracy. In addition we prove that the rewrite can be extended to transformer architectures.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：非结构化剪枝（unstructured pruning）虽能大幅减少有效参数数（mask-alive count），但标准实现（如 PyTorch `torch.nn.utils.prune`）保持权重张量形状不变，部署时仍需运行全尺寸密集核，导致实际模型大小和推理速度并未降低。报告中的 mask-alive count 不能反映可部署模型的真实大小，与用户关心的部署尺寸之间存在巨大差距。
- **背景**：结构化剪枝虽能直接缩小网络，但非结构化剪枝通常能达到更高的压缩-精度权衡。然而，通用 GPU 对任意稀疏模式的支持有限，速度提升很小。现有工作大多忽略“实际移除参数”这一结构重写问题，或仅做近似。
- **本文目标**：提出一种精确的结构重写（称为最小化 Minimization），将任意非结构化剪枝后的掩码网络转换为更小的密集网络，且前向函数在浮点误差内完全一致。并进一步通过迭代循环（Squeeze-Release）反复利用被浪费的容量，达到比单次剪枝+最小化更小的可部署网络。

### 2. 论文提出的方法论

- **核心思想**：将剪枝分解为两个独立问题——“决定剪哪些参数”和“实际移除它们”。本文聚焦后者，提出精确结构最小化（Section 3.2），并嵌入一个四步循环：prune → squeeze（最小化） → release（释放） → fine-tune。
- **关键技术细节**：

  - **最小化（Squeeze）**：移除死亡输入单元（dead-incoming）与死亡输出单元（dead-outgoing）。对于全连接层：若某行权重全零，则该神经元输出为常数，可将该常数折叠到下游偏置中，然后删除对应行和列；若某列全零，则直接删除行列。对于 Conv2d，只能整个输出滤波器全零时才能删除。该重写保持函数精确。
  - **CompensatedLayerNorm（补偿层归一化）**（Section 3.3）：在 LayerNorm 中，删减通道会改变所有幸存通道的均值和方差。本文提出一种函数保持的替换方案：存储被删通道的三个充分统计量（数量 K、和 S、平方和 Q），在向前传播时重建全宽度的均值和方差，使幸存通道输出与原始 LayerNorm 完全一致。仅需额外存储三个标量。
  - **释放（Release）**：最小化后，压缩张量中仍存在许多零（由于行列未全零而无法删除）。将这些位置用服从 `N(μ, σ²)·0.01` 的小噪声初始化，重新变为可训练参数，而不改变张量形状和参数计数。后续微调可以通过梯度下降利用这些容量。
  - **迭代循环**：prune（基于梯度×权重大小的重要性分数，并引入回滚机制）→ squeeze（最小化）→ release（释放零位）→ fine-tune。重复直到网络无法再最小化或达到最大循环数。对比基线（No-minimize）只做 prune+fine-tune，不做 squeeze 和 release，最终再应用一次后验最小化。

### 6. 论文的主要结论与发现

- **精确最小化是通用工具**：对任意非结构化剪枝后的模型，进行一次后验最小化即可大幅缩小可部署模型（FC 上 10.5×，ConvNeXt 上 2.78×，ViT 上 1.12×），且保持函数精确。
- **Squeeze-Release 循环进一步压缩**：在 FC 上达到 39.2× 缩小（与未剪枝模型比），在 ConvNeXt-Tiny 上达到 14.8× 缩小，且精度相当。
- **Release 步骤是关键**：Squeeze-Release 循环数远超 No-minimize 基线（ConvNeXt 上从 3.6 次增至 97+ 次），表明释放的零位被重新训练为有用参数，使得后续剪枝能发现新结构冗余。
- **CompensatedLayerNorm**：使得在 LayerNorm 残差流中精确删减通道成为可能，额外带来 1.3× 压缩增益，且存储开销仅为 3 个标量。
- **mask-alive count 与可部署尺寸存在巨大差距**：文中量化了这一差距（FC 5.4×，ConvNeXt 11.3×，ViT 5.0×），建议社区同时报告 mask-alive 和后最小化尺寸。

### 7. 优点

- **理论严谨**：最小化是精确函数保持的，CompensatedLayerNorm 也是精确重建，甚至经过浮点误差验证（最大绝对 logit 差 ≤ 1.06×10⁻⁶）。
