# Abstract 

Existing video coding for machines is often trained for a specific downstream task and model. As a result, the compressed representation becomes tightly coupled to the end task, making it difficult to scale across multiple tasks or adapt to model updates. We propose PAT-VCM, a plug-and-play auxiliary-token framework for video coding for machines. PAT-VCM keeps a shared baseline compressed stream and augments it with lightweight task-aware auxiliary tokens, allowing different downstream tasks to recover the information they need without retraining a separate codec for each task. The framework supports three forms of auxiliary information: visual residual tokens, prompt/control tokens, and semantic tokens. We evalu-ate PAT-VCM on segmentation, depth estimation, and semantic recognition. A shared detection-oriented auxiliary branch provides a reusable first refinement, task-specific visual branches improve segmentation and depth, prompt tokens provide further segmentation gains at negligible bitrate, and semantic tokens achieve strong recognition performance with extremely low overhead. These results suggest that a shared compressed representation, combined with lightweight task-aware auxiliary tokens, is a practical and scalable alternative to tightly task-coupled VCM design. 

# 1 Introduction 

Learned visual compression has made steady progress through end-to-end optimized autoencoders, hierarchical priors, autoregressive entropy models, and neural video codecs [ 2– 4, 17 , 19 ]. More recently, tokenized and generative approaches have started to represent video with discrete or latent tokens decoded by strong generative models [ 1, 29 ]. These methods improve compactness and reconstruction quality, but their main objective is still visual reconstruction for human viewing. In many deployment settings, compressed video is consumed not only by humans but also by machine models for detection, segmentation, depth estimation, and semantic recognition. This has led to growing interest in coding for machines, task-oriented compression, and joint human–machine coding, where the goal is not only to preserve pixel quality but also to preserve downstream task performance [ 7, 13 , 26 , 28 ]. In this setting, the question is not just how well a bitstream reconstructs pixels, but whether it preserves the information needed by different machine tasks under a limited bitrate. Most machine-oriented compression methods still assume that one compressed representation can support all downstream tasks. In practice, this is difficult for two reasons. First, different tasks rely on different information. Detection depends on object-level localization cues. Segmentation is more sensitive to boundaries and decoder interaction. Depth estimation depends on local geometric structure. Recognition depends more on preserving semantic features. A single token stream is therefore unlikely to preserve all of these equally well. Second, current VCM methods are often trained for a specific downstream task model, which makes the compressed representation tightly coupled to the end task used during training. Such a design is hard to scale: each new task may

require retraining a new codec, and even different downstream models for the same task may favor different task-specific representations. As a result, it is difficult to support heterogeneous machine tasks with one shared compressed stream. This motivates the main idea of our paper. Instead of forcing one compressed representation to serve all downstream tasks, we start from a shared baseline codec and augment it with auxiliary tokens .These tokens are task-aware additions to the baseline bitstream. They are used to recover or preserve information that is important for a downstream task but not well retained in the shared compressed representation. Different auxiliary streams can have different token types, including visual residual tokens, prompt/control tokens, and semantic tokens. Based on this idea, we propose PAT-VCM , a plug-and-play auxiliary token codec for video coding for machines. PAT-VCM consists of a shared baseline codec and attachable auxiliary token streams. The baseline codec produces a primary compressed stream and a coarse reconstructed video. Auxiliary streams are added only when needed for a downstream task. This keeps the baseline representation shared, while allowing task-specific information to be introduced in a lightweight and modular way. A key feature of PAT-VCM is that the auxiliary design is plug-and-play. The auxiliary streams are attached to the baseline codec, rather than requiring a separate codec for each task. The downstream models remain frozen and operate on refined decoded video or decoder-side prompts. This keeps the method at the codec level and avoids retraining the full system whenever the downstream task or model changes. We evaluate PAT-VCM on several downstream tasks with different requirements. A shared detection-oriented auxiliary stream improves localization from compressed video and provides a common first refinement. On top of it, a segmentation-specific visual stream and a compact point-prompt codebook improve SAM-based mask prediction [ 15 , 24 ]. A depth-specific visual stream improves monocular depth estimation under a frozen depth backbone [ 27 ]. We also extend the framework to semantic recognition with a frozen CLIP backbone [ 22 ]. These results show that a shared compressed representation can be made much more useful for heterogeneous machine tasks by adding lightweight task-aware auxiliary tokens. Our contributions are as follows: • We introduce PAT-VCM, a plug-and-play auxiliary token codec for video coding for ma-chines, built on a shared baseline compressed stream with attachable task-aware auxiliary token streams. • We present a unified auxiliary-token view of machine-oriented compression, covering multiple token types including visual residual tokens, prompt/control tokens, and semantic text tokens. • We show that a shared detection-oriented auxiliary stream can serve as a reusable early refinement across downstream tasks, while later auxiliary streams remain task-specific. • We validate the framework on representative tasks including segmentation, depth estimation, and semantic recognition, and show that different auxiliary token types address different downstream bottlenecks under modest bitrate overhead. 

# 2 Related Work 

2.1 Learned, Tokenized, and Generative Visual Compression 

Learned image and video compression has progressed through end-to-end optimized transform coding, hierarchical priors, autoregressive entropy models, and neural predictive video codecs [ 2 –4, 17 , 19 ]. More recently, tokenized and generative codecs have explored discrete or latent visual representations decoded by strong neural generators, shifting part of reconstruction from hand-designed predictive structure to learned priors over visual content [ 1, 8 ]. These methods provide increasingly compact and expressive compressed representations, but remain centered on reconstruction quality, whether measured by distortion or perceptual realism. Our work starts from the same tokenized-compression setting, but asks a different question: how should a shared compressed representation be augmented when decoded video is primarily consumed by downstream machine models? 22.2 Coding for Machines and Task-Oriented Compression 

Coding for machines and task-oriented compression aim to preserve downstream task performance under rate constraints rather than optimizing only human perceptual quality [ 7, 26 ]. Existing work has explored feature-preserving compression, semantic communication, task-aware image coding, and joint human–machine coding, typically targeting one downstream task or one shared machine representation [ 13 , 28 ]. Our work is aligned with this motivation, but differs in formulation. Instead of learning one task-specific codec or assuming that one compressed representation should serve all downstream tasks equally well, PAT-VCM introduces a shared baseline codec together with task-aware auxiliary token streams. This shifts the focus from single-task optimization to modular support for heterogeneous downstream tasks. 

2.3 Frozen Downstream Models and Promptable Decoders 

Recent vision models provide strong frozen models for a range of downstream tasks. DETR offers an end-to-end transformer formulation for object detection [ 5]. Promptable segmentation models such as SAM and SAM 2 expose decoder-side conditioning through boxes and points, making them especially relevant for studying control signals under compression [ 15 , 24 ]. For geometry, MiDaS and Depth Anything provide robust monocular depth estimators that generalize well across datasets [23 , 27 ]. Our PAT-VCM is designed for this setting: the downstream models remain frozen, while the codec is augmented with auxiliary tokens that refine either the decoded video itself or the control signal presented to the downstream decoder. 

2.4 Vision–Language Models and Semantic Recognition 

Vision–language models (VLMs) provide a useful setting for semantic recognition under compression. CLIP and related contrastive VLMs learn transferable image-text embeddings that support zero-shot recognition and open-vocabulary matching [ 22 ]. This makes them a natural frozen semantic model for testing whether compression preserves task-relevant semantics. In PAT-VCM, recognition extends machine-oriented compression beyond detection and prediction, and evaluates whether auxiliary tokens can preserve semantic information under a shared compressed representation. 

2.5 Shared Representations and Task-Specific Adaptation 

Multi-task learning often combines a shared representation with lightweight task-specific adaptation [ 6]. Similar ideas appear in residual adapters for vision, where a shared backbone is augmented by small task-dependent modules [ 25 ], and in prompt-based adaptation, where task-specific control is introduced without changing the backbone itself [ 14 ]. Our setting is related, but the adaptation happens at the codec level rather than inside the downstream model. In PAT-VCM, the baseline compressed stream serves as the shared representation, while auxiliary tokens provide task-specific correction or control for different tasks. 

# 3 Method 

3.1 PAT-VCM Formulation 

The basic observation is that one compressed representation is rarely sufficient for many downstream machine tasks. Detection, segmentation, depth estimation, and recognition do not rely on exactly the same information. A bitstream that is good for general reconstruction may still lose cues that matter for a particular task. PAT-VCM addresses this by separating the codec into two parts: a shared baseline stream and task-aware auxiliary streams (as illustrated in Figure 1). The baseline codec produces a compact primary representation and a coarse reconstructed video that are shared across tasks. Auxiliary streams are then attached only when a downstream task needs additional information that is not preserved well enough by the baseline representation. Let x ∈ RB×c×T ×H×W denote an input video, where B, c, T are the batch size, number of channels, and number of frames, and H and W are the spatial resolution. Lbase = Lrecon + βLrate + λLquant ,

where Lrecon measures reconstruction error, Lrate penalizes bitrate, and Lquant is the regularization associated with quantization. After training, the baseline codec is frozen and used as a shared compressed representation. For a downstream task t, PAT-VCM introduces a task-specific auxiliary stream at. In the most common case, this stream is decoded into a pixel-space residual: 

rt = Gt(at, ˆx0), ˆxt = ˆ x0 + rt,

where Gt is a task-specific refinement decoder and ˆxt is the refined video presented to the downstream model. A frozen downstream model ft then produces 

yt = ft(ˆ xt).

Not all auxiliary information has to act through pixel refinement. Depending on the task, an auxiliary stream may be decoded into a pixel residual, used as a prompt or control signal, or transmitted directly as a compact semantic token. In this sense, PAT-VCM supports auxiliary adaptation at three levels: pixel refinement, decoder control, and semantic result transmission. The key point is that the primary stream z0 remains shared, while the auxiliary stream carries task-specific information. A deliberate design choice in PAT-VCM is that the downstream models remain frozen. This keeps the adaptation at the codec level. It also avoids retraining a new codec together with every new downstream model. This is important in the long run because downstream models evolve quickly in practice: if each task or model update required codec retraining, deployment and maintenance would become expensive and difficult to scale. 

3.2 Auxiliary Token Types 

PAT-VCM supports different types of auxiliary tokens because different downstream tasks lose different kinds of information under compression. The first type is a visual auxiliary stream . These tokens are decoded into a pixel-space residual and are used when the task suffers from missing local detail, degraded boundaries, or weakened region-level structure. 4The second type is a prompt/control stream . These tokens do not primarily change the pixels. Instead, they modify how a frozen downstream decoder is queried. The point-prompt codebook in the segmentation setting is an example of such tokens. The third type is a semantic token stream . These tokens transmit lightweight task-level outputs computed by the encoder on the original frame, rather than pixel residuals. They are useful when the downstream goal is better served by compact semantic results than by further pixel refinement. In the current framework, semantic tokens are used for recognition through a class-label token, and can also be extended to structured outputs such as pose. These three token types operate at different levels. Visual tokens improve the reconstructed input, prompt/control tokens improve decoder interaction, and semantic tokens transmit task-relevant information directly. 

3.3 Representative Task Families 

All task-specific visual branches share the same basic design: an ROI is processed by an auxiliary encoder, quantized, and decoded into a pixel-space residual that is written back to the reconstructed video. The architecture is shared across tasks, and only the supervision changes. This keeps the framework modular, since adding a new visual task does not require a new auxiliary network, only a frozen downstream model and a task-specific loss. PAT-VCM is meant to be a general auxiliary-token framework rather than a task-specific pipeline. We therefore study several representative task families built on the same shared baseline codec. 

Detection. Detection provides the shared early refinement in the current system. Starting from the coarse reconstructed video ˆx0, the codec refines ROI regions that are likely to affect localization and feeds the refined video to the same frozen detector. After training, this detection-oriented branch is frozen and reused by all later task-specific branches. 

Segmentation. For segmentation, we use a task-specific visual auxiliary branch together with an optional prompt branch. The visual branch is trained to improve the input seen by a frozen SAM-based segmentor. We further use a compact point-prompt codebook to refine decoder interaction. In the current design, the codebook contains 32 candidate point locations, so transmitting one selected point requires 5 bits per ROI, while a foreground-background pair requires 10 bits per ROI. This setting combines two complementary mechanisms: visual refinement improves the reconstructed input, while prompt refinement improves the decoder query. 

Depth estimation. Depth estimation is a dense geometric prediction task. As in segmentation, the same auxiliary architecture is applied on ROI regions, but with a different supervision signal. The depth-specific auxiliary branch is trained so that the refined video produces depth predictions closer to those from the original frame under a frozen monocular depth model. This shows that the same auxiliary-token framework extends beyond mask prediction to absolute per-pixel regression. 

Semantic recognition. Recognition targets semantic preservation rather than dense pixel prediction. PAT-VCM can use either a visual auxiliary branch or a semantic token branch for the task. The visual branch refines object ROI regions to better preserve CLIP features used by a frozen vision encoder. The semantic token branch transmits lightweight task-level outputs, such as a class-label token, directly from the encoder. This setting shows that the framework is not limited to pixel refinement, and that semantic information can also be conveyed through very low-rate auxiliary tokens. 

# 4 Experiments 

4.1 Experimental Setup Baseline compressed video. All experiments start from the same compressed video generated by the baseline tokenizer. We use the Cosmos DV4x8x8 tokenizer and reconstruction model from the Cosmos World Foundation Model platform [ 1 ] as the baseline codec. The downstream task output obtained from this compressed video is denoted as Cosmos . For reference, we also report the 5downstream output obtained from the uncompressed original video, denoted as Orig . The baseline tokenizer remains frozen throughout all experiments. 

Downstream models. For detection, we use a frozen Deformable DETR detector [ 5]. For segmen-tation, we use a frozen SAM-based segmentor [ 15 , 24 ]. For depth estimation, we use a frozen Depth Anything V2 Small model [ 27 ]. For semantic recognition, we use a frozen CLIP ViT-B/32 vision encoder [22]. 

Auxiliary branches. The task-specific visual auxiliary streams use the same core design: an auxiliary encoder, a finite scalar quantizer, and a residual decoder that predicts a pixel-space correction to the compressed video. In the current implementation, the auxiliary encoder uses 256 hidden channels, the FSQ module uses levels [8 , 8, 8, 8] , and the residual decoder uses 128 hidden channels. Each task-specific module has about 5.5M trainable parameters. The shared detection-oriented branch, denoted as Det-Aux , is applied before the later task-specific branches. On top of Det-Aux, we instantiate a segmentation-specific branch ( Seg-Aux ), a depth-specific branch ( Depth-Aux ), and a recognition-oriented branch for CLIP feature preservation (CLIP-Aux ). For segmentation, we also evaluate a point-prompt codebook as a prompt/control stream. For recognition, we further evaluate a semantic-token form in which a lightweight task-level result is transmitted directly. 

Datasets and metrics. For segmentation, we evaluate on DAVIS [ 20 , 21 ], MOSE [ 9 ], and VIPSeg [18]. The main metric is mean IoU, together with Recall@0.5 and Recall@0.75 [12, 16]. For depth estimation, we evaluate on DAVIS and VIPSeg. Because monocular depth is defined only up to scale and shift, we use per-frame scale–shift least-squares alignment and report AbsRel, RMSE, and δ < 1.25 , following standard monocular depth evaluation practice [ 10 , 11 ]. We report both full-frame and ROI-only depth metrics. For semantic recognition, we evaluate CLIP embedding preservation through class agreement with the original-frame prediction. Since the semantic-token branch directly transmits a lightweight task-level result, we also report its bits per ROI. 

4.2 Shared Detection-Oriented Auxiliary Stream 

We begin with the shared detection-oriented branch, Det-Aux. Its role is to recover object-level cues weakened by compression and to provide stronger region hypotheses for later tasks. On DAVIS, Det-Aux improves detection recall by 0.085 over the Cosmos baseline. On the subset of hard detection cases, defined as objects whose detection IoU under the Cosmos baseline is below 0.30 , it further improves IoU by 11 .1 percentage points. These results show that Det-Aux is useful not only for detection itself, but also as a reusable first refinement for later task-specific branches. 

4.3 Segmentation Segmentation-specific visual refinement. We first evaluate a task-specific visual branch, Seg-Aux, on top of Det-Aux. The downstream segmentor remains frozen, so Seg-Aux can improve segmentation only by improving the visual evidence seen by the segmentor. 

Prompt/control tokens. For segmentation, we further use a compact prompt/control stream. A prompt token is a discrete index into a fixed codebook of candidate point locations inside the Det-Aux ROI box. The codebook contains 32 candidates, constructed from a 6 × 6 grid with the four corners removed. At the encoder, where the original frame is available, we first compute a target mask by running SAM-huge (641M parameters) [ 15 ] on the original frame with the ROI box. We then evaluate all candidate points in the codebook by running SAM-base (93.7M parameters) [ 15 ] on the refined compressed frame using the same ROI box and the candidate point as a prompt, and comparing the resulting mask with the target mask from the original frame. The candidate that achieves the highest mask IoU is selected, and only its codebook index is transmitted. At the decoder, this index is mapped back to a point location inside the ROI box and used as a prompt for SAM-base. A one-point prompt therefore requires only 5 bits per ROI. The main loss is a scale-invariant log-depth loss on ROI regions, together with an L1 regularizer between the refined ROI and the Det-Aux ROI. The pseudo-ground-truth depth is obtained by applying the frozen depth model to the original frame. Table 2 reports the DAVIS results. Depth-Aux is highly effective, reducing ROI AbsRel from 3.764

Compared with the compressed baseline, Depth-Aux recovers more coherent object-level depth structure and reduces large local errors inside ROI regions. Several examples are marked with “(>Orig)”, indicating that the refined compressed representation yields better depth estimation than the uncompressed original under the same frozen Depth Anything V2 evaluation protocol.

Overall, these analyses show that the largest gains come from difficult cases and larger ROIs, while easy cases change only modestly. 

4.5 Semantic Recognition Visual auxiliary branch. A recognition-oriented visual branch can be trained to preserve CLIP features on object ROIs. In this case, the auxiliary branch refines the crop so that the CLIP embedding of the refined crop remains close to that of the original crop. This gives a modest but consistent gain over the baseline. 

Semantic tokens. Recognition also uses a more compact semantic-token form. This should be viewed as an upper-end case of the framework: given an object ROI from the upstream localization stage, the encoder transmits a lightweight task-level semantic result directly, rather than further refining pixels. In the current implementation, the encoder applies the frozen CLIP vision encoder to the ROI crop from the original frame and compares the resulting feature with a fixed set of text embeddings for the candidate object classes. The predicted class is then encoded as a discrete class-label token and transmitted. Since the current setup uses 80 candidate classes, the label can be represented with 7 bits per ROI. At the decoder, this token is interpreted directly as the recognized class for that ROI, without requiring CLIP inference on the compressed crop. The result therefore measures semantic preservation conditioned on localization, not joint detection and recognition. Table 3 shows that this semantic token largely outperforms visual refinement while requiring only 7

bits per ROI. This suggests that when the downstream target is already semantic, compact task-level tokens can preserve task performance much more efficiently than additional pixel refinement.

4.7 Additional Tasks in the Appendix 

In addition to the main tasks in the paper, we report several additional task studies in the appendix, including surface normal estimation and pose estimation. These studies help clarify both the broader applicability and the current limits of the framework. In particular, pose estimation shows that semantic tokens can also support structured outputs, while surface normal estimation shows that some tasks benefit less under the current ROI-based residual design. These results further support the main point of the paper: different downstream tasks may call for different forms of auxiliary information. 94.8 Limitations 

The current PAT-VCM design is most effective when the downstream task can benefit from local ROI refinement or compact task-level semantic outputs. Tasks whose predictions depend more strongly on spatial derivatives or cross-boundary consistency, such as surface normals under the current ROI residual formulation, show much smaller gains. In addition, although PAT-VCM avoids retraining a separate end-to-end codec for every downstream model, task-specific auxiliary branches still need task-specific supervision. Extending the framework toward more unified auxiliary learning across tasks is an important direction for future work. 

# 5 Conclusion 

We presented PAT-VCM, a plug-and-play auxiliary-token framework for video coding for machines. PAT-VCM keeps a shared baseline compressed stream and augments it with lightweight task-aware auxiliary tokens, instead of training a separate codec for each downstream task and model. This reduces coupling to any single end task and makes the framework easier to reuse across tasks and model updates. Our results show that this design is effective across different task types. A shared detection-oriented auxiliary branch provides a reusable first refinement. Task-specific visual branches improve seg-mentation and depth estimation. Prompt/control tokens provide additional gains for segmentation at negligible bitrate, while semantic tokens are particularly effective for recognition. Taken together, these results show that a shared compressed representation, combined with lightweight auxiliary tokens, can support heterogeneous machine tasks more flexibly than tightly task-specific VCM design.

# D Additional Task Study: Surface Normals 

This appendix reports an additional task study that helps clarify the current scope of PAT-VCM under the ROI-based residual design. Unlike segmentation and depth estimation, surface normal estimation shows only marginal gains. We evaluate surface normals as a secondary geometric task. The normal-specific auxiliary branch uses the same auxiliary architecture as the main task-specific branches, 

AuxEncoder-256 + FSQ[8,8,8,8] + ResidualDecoder-128 ,

and is trained on ROI regions with a cosine similarity loss on surface normals. In our setup, normals are derived from depth predictions through spatial gradients, so the task depends on local surface orientation rather than absolute depth values. 

Results. Table 12 summarizes the ROI normal results on DAVIS. The normal-specific branch gives only a marginal improvement over the shared Det-Aux baseline. In addition, the depth-specific branch slightly degrades normal quality, even though it improves absolute depth estimation. 

Discussion. These results suggest that the current ROI residual design is less effective for gradient-derived targets than for absolute per-pixel predictions such as depth. A likely reason is that surface normals depend on spatial derivatives, so small discontinuities at ROI boundaries may have limited effect on depth values but noticeably affect local orientation. 14 Table 12: Surface normal estimation on DAVIS ROI regions. # E Additional Task Study: Pose Estimation 

We also evaluate human pose estimation as an additional structured prediction task. The downstream model is a frozen Keypoint R-CNN with a ResNet50-FPN backbone. In the visual-auxiliary setting, the pose branch uses the same auxiliary architecture as the segmentation and depth branches and is trained by feature distillation on the frozen Keypoint R-CNN backbone within person ROIs. In addition to this visual branch, pose also admits a semantic-token form in which compact keypoint outputs are transmitted directly. The detailed architecture and training setup follow the same shared auxiliary design used in the main tasks. 

Baseline compression gap. Pose estimation has a clear compression gap. In the baseline evaluation, Cosmos gives a mean keypoint error (MKE) of 28 .0 pixels on DAVIS and 20 .4 pixels on VIPSeg, while the shared Det-Aux branch does not consistently improve this task. On DAVIS, the baseline error corresponds to about 5.5% of the frame width, indicating that compression noticeably affects fine-grained joint localization. 

Semantic skeleton tokens. For pose, we further evaluate a semantic-token form in which quantized keypoints are transmitted directly. In the current implementation, the skeleton token uses an 8 × 8

quantization grid and requires 102 bits per person. Table 13 summarizes the DAVIS results. The skeleton token reduces MKE from 33 .62 pixels for the Cosmos baseline to 7.33 pixels, corresponding to a 78% reduction. Although this token is larger than the class-label token used for recognition, it is still negligible compared with the ROI pixel auxiliary stream. Table 13: Pose estimation on DAVIS.
