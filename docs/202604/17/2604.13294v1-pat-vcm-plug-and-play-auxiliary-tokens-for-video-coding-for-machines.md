## 摘要
现有的面向机器的视频编码（VCM）通常针对特定的下游任务和模型进行训练。因此，压缩表示与最终任务紧密耦合，导致其难以扩展到多个任务或适应模型更新。我们提出了 PAT-VCM，一种用于机器视频编码的即插即用辅助 Token 框架。PAT-VCM 保留了一个共享的基准压缩流，并使用轻量级的任务感知辅助 Token 对其进行增强，从而允许不同的下游任务恢复所需信息，而无需为每个任务重新训练单独的编解码器。该框架支持三种形式的辅助信息：视觉残差 Token、提示/控制 Token 以及语义 Token。我们在分割、深度估计和语义识别任务上对 PAT-VCM 进行了评估。实验结果显示，一个共享的面向检测的辅助分支提供了可重用的初步细化，特定任务的视觉分支提升了分割和深度估计性能，提示 Token 在极低码率下进一步提高了分割增益，而语义 Token 则以极低的开销实现了强大的识别性能。这些结果表明，共享压缩表示结合轻量级任务感知辅助 Token，是替代紧耦合任务 VCM 设计的一种实用且具扩展性的方案。

## Abstract
Existing video coding for machines is often trained for a specific downstream task and model. As a result, the compressed representation becomes tightly coupled to the end task, making it difficult to scale across multiple tasks or adapt to model updates. We propose PAT-VCM, a plug-and-play auxiliary-token framework for video coding for machines. PAT-VCM keeps a shared baseline compressed stream and augments it with lightweight task-aware auxiliary tokens, allowing different downstream tasks to recover the information they need without retraining a separate codec for each task. The framework supports three forms of auxiliary information: visual residual tokens, prompt/control tokens, and semantic tokens. We evaluate PAT-VCM on segmentation, depth estimation, and semantic recognition. A shared detection-oriented auxiliary branch provides a reusable first refinement, task-specific visual branches improve segmentation and depth, prompt tokens provide further segmentation gains at negligible bitrate, and semantic tokens achieve strong recognition performance with extremely low overhead. These results suggest that a shared compressed representation, combined with lightweight task-aware auxiliary tokens, is a practical and scalable alternative to tightly task-coupled VCM design.

*   **研究动机**：在实际应用中，同一段视频可能被多种机器视觉模型消费。作者希望构建一种“即插即用”的框架，通过一个**共享的基准压缩流**，配合**轻量级的任务感知辅助 Token**，实现对多种异构任务的高效支持。

### 2. 方法论
*   **核心思想**：将编解码器分为两部分：
    1.  **共享基准流（Shared Baseline Stream）**：提供通用的压缩表示和粗糙的重建视频。
    2.  **辅助 Token 流（Auxiliary Token Streams）**：根据具体任务需求，动态挂载轻量级的辅助信息。
*   **关键技术细节**：
    *   **三种辅助 Token 类型**：
        *   **视觉残差 Token**：在像素空间提供残差修正，用于恢复局部细节和边界。
        *   **提示/控制 Token（Prompt/Control Tokens）**：不改变像素，而是作为下游模型（如 SAM）的输入提示（如点、框坐标），引导模型解码。
        *   **语义 Token**：直接传输任务级结果（如类别标签、骨架关键点），绕过像素重建。
    *   **共享检测分支（Det-Aux）**：首先训练一个面向检测的辅助分支，用于细化目标区域（ROI），该分支被冻结并作为后续所有任务（分割、深度等）的通用预处理步骤。
    *   **算法流程**：输入视频 $\rightarrow$ 基准编码器 $\rightarrow$ 基础码流 + 粗糙重建 $\rightarrow$ 任务特定辅助编码器（针对 ROI） $\rightarrow$ 辅助码流 $\rightarrow$ 辅助解码器 $\rightarrow$ 细化后的视频/提示信息 $\rightarrow$ 冻结的下游模型。

### 3. 实验设计
*   **数据集**：DAVIS、MOSE、VIPSeg（用于分割和深度估计）；DAVIS（用于语义识别和姿态估计）。
*   **Benchmark 与对比对象**：
    *   **Orig**：未压缩的原始视频。
    *   **Cosmos**：基于 Cosmos 基础模型的基准压缩方案（无辅助信息）。
    *   **对比配置**：Cosmos vs. Det-Aux vs. 任务特定辅助分支（Seg-Aux, Depth-Aux 等）。
*   **下游模型（均处于冻结状态）**：Deformable DETR（检测）、SAM/SAM 2（分割）、Depth Anything V2（深度）、CLIP（识别）、Keypoint R-CNN（姿态）。

### 4. 资源与算力
*   **硬件环境**：使用单张 **NVIDIA Tesla V100-SXM2-32GB GPU** 进行推理测试。
*   **算力开销**：
    *   基准解码（Cosmos）耗时约 773.7ms/clip。

*   **模型规模**：每个任务特定模块约有 **5.5M** 可训练参数。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了分割、深度估计、语义识别、表面法线估计、姿态估计等 **6 种任务**。
*   **消融实验**：针对 Token 类型（视觉 vs. 提示）、ROI 大小敏感度、任务难度（Easy/Medium/Hard 案例）进行了详细分析。
*   **充分性评价**：实验设计较为全面，不仅验证了性能提升，还分析了码率开销和推理延迟。通过在多个标准数据集上测试，证明了框架的泛化能力。

### 6. 主要结论与发现
*   **模块化有效性**：通过添加极少量的辅助码率（如 5-10 bits 的提示 Token），可以显著弥补压缩带来的性能损失。
*   **超越原始视频**：在某些情况下（如分割和深度估计），经过辅助分支细化的压缩视频表现甚至**优于原始未压缩视频**，因为辅助分支起到了针对性的增强作用。
*   **语义 Token 的高效性**：对于识别任务，直接传输 7 bits 的类别 Token 比进行像素级修复更高效。
*   **共享分支的价值**：Det-Aux 作为通用的第一层细化，对后续多种任务均有正面贡献。

### 7. 优点
*   **解耦与扩展性**：实现了编解码器与下游模型的解耦，支持“即插即用”，极大降低了多任务部署的成本。
*   **多模态支持**：统一了视觉残差、控制信号和语义标签的传输框架。
*   **码率效率**：利用 ROI 编码和离散 Token，在极低码率增量下实现了显著的性能增益。

### 8. 不足与局限
*   **ROI 依赖性**：目前的性能提升主要集中在 ROI 区域，对于需要全局上下文或空间导数信息的任务（如表面法线估计）效果有限。
*   **编码端复杂度**：某些辅助 Token（如点提示）的生成需要在编码端进行多次模型推理，增加了编码端的计算负担。
*   **监督需求**：虽然编解码器是通用的，但每个辅助分支仍需要针对特定任务的监督信号进行训练。
