## 摘要
现有的面向机器的视频编码通常针对特定的下游任务和模型进行训练。因此，压缩表示与最终任务紧密耦合，导致其难以扩展到多个任务或适应模型更新。我们提出了 PAT-VCM，一种用于面向机器视频编码的即插即用辅助 Token 框架。PAT-VCM 保留了一个共享的基准压缩流，并使用轻量级的任务感知辅助 Token 对其进行增强，使不同的下游任务能够恢复所需信息，而无需为每个任务重新训练单独的编解码器。该框架支持三种形式的辅助信息：视觉残差 Token、提示/控制 Token 以及语义 Token。我们在分割、深度估计和语义识别任务上对 PAT-VCM 进行了评估。实验结果显示，一个共享的面向检测的辅助分支提供了可重用的初步细化，特定任务的视觉分支提升了分割和深度估计性能，提示 Token 在码率开销极小的情况下提供了进一步的分割增益，而语义 Token 则以极低的开销实现了强大的识别性能。这些结果表明，共享压缩表示结合轻量级任务感知辅助 Token，是替代任务紧耦合 VCM 设计的一种实用且具扩展性的方案。

## Abstract
Existing video coding for machines is often trained for a specific downstream task and model. As a result, the compressed representation becomes tightly coupled to the end task, making it difficult to scale across multiple tasks or adapt to model updates. We propose PAT-VCM, a plug-and-play auxiliary-token framework for video coding for machines. PAT-VCM keeps a shared baseline compressed stream and augments it with lightweight task-aware auxiliary tokens, allowing different downstream tasks to recover the information they need without retraining a separate codec for each task. The framework supports three forms of auxiliary information: visual residual tokens, prompt/control tokens, and semantic tokens. We evaluate PAT-VCM on segmentation, depth estimation, and semantic recognition. A shared detection-oriented auxiliary branch provides a reusable first refinement, task-specific visual branches improve segmentation and depth, prompt tokens provide further segmentation gains at negligible bitrate, and semantic tokens achieve strong recognition performance with extremely low overhead. These results suggest that a shared compressed representation, combined with lightweight task-aware auxiliary tokens, is a practical and scalable alternative to tightly task-coupled VCM design.

*   **核心问题**：传统的面向机器的视频编码（VCM）通常针对特定任务（如检测或分割）和特定模型进行端到端训练。这导致压缩表示与下游任务高度耦合，缺乏扩展性。当任务增加或下游模型更新时，往往需要重新训练整个编解码器，部署成本极高。
*   **研究动机**：为了解决“一任务一模型一编码”的不可扩展性，作者提出了一种模块化方案，旨在通过一个共享的基准压缩流，配合轻量级的“即插即用”辅助 Token，来支持多种异构的机器视觉任务。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：将编解码器解耦为“共享基准流”和“任务感知辅助流”。基准流提供通用的粗糙重建，辅助流则根据具体任务需求补充关键信息。
*   **关键技术细节**：
    *   **共享基准编解码器**：使用预训练且冻结的 Tokenizer（如 Cosmos）生成主比特流。
    *   **三类辅助 Token**：
        1.  **视觉残差 Token (Visual Residual Tokens)**：针对感兴趣区域（ROI）生成像素级的修正残差，用于恢复局部细节和边缘。
        2.  **提示/控制 Token (Prompt/Control Tokens)**：不改变像素，而是作为下游模型（如 SAM）的输入提示（如点、框坐标），以极低码率引导模型预测。
        3.  **语义 Token (Semantic Tokens)**：直接传输高层语义结果（如类别标签、骨架关键点），跳过像素重建。
    *   **分层细化架构**：引入一个共享的“面向检测的辅助分支（Det-Aux）”，作为所有任务的通用初步细化步骤，随后再挂载特定任务的辅助分支。
*   **算法流程**：输入视频 $\rightarrow$ 基准编码 $\rightarrow$ 粗糙重建 $\rightarrow$ Det-Aux 细化 $\rightarrow$ 任务特定辅助 Token 增强 $\rightarrow$ 冻结的下游模型输出结果。

*   **消融实验**：针对 Token 类型（视觉 vs 提示）、ROI 大小、任务难度（易、中、难案例）进行了详细的对比分析。
*   **充分性评价**：实验设计较为充分且客观。通过在多个数据集上验证，并对比原始视频性能，证明了该框架在极低额外开销下能显著弥补压缩损失，甚至在某些情况下超过原始视频的机器处理效果（由于针对性的 ROI 增强）。

### 6. 主要结论与发现
*   **共享分支的有效性**：面向检测的辅助分支（Det-Aux）可以作为多种下游任务的通用预处理步骤。
*   **提示 Token 的高效性**：在分割任务中，仅需 5-10 bits 的提示 Token 即可获得比视觉残差更大的性能提升。
*   **语义 Token 的优势**：对于识别任务，直接传输类别标签 Token 比进行像素细化更高效。
*   **性能跨越**：PAT-VCM 成功打破了 VCM 的紧耦合限制，实现了在不重新训练主编解码器的情况下，灵活支持多种机器视觉任务。

### 7. 优点
*   **高扩展性**：即插即用设计，支持下游模型和任务的快速迭代。
*   **多维适配**：同时支持像素级、提示级和语义级的辅助信息，覆盖了不同任务的需求。
*   **码率效率**：辅助 Token 带来的码率增加相对较小，尤其是提示和语义 Token 几乎可以忽略不计。

### 8. 不足与局限
*   **ROI 依赖性**：目前的视觉残差设计基于 ROI，对于需要全局上下文或跨边界一致性的任务（如表面法线估计）提升有限。
*   **推理开销**：虽然码率低，但每增加一个辅助分支都会增加一定的计算延迟。
*   **监督需求**：虽然不需要重训编解码器，但每个新的辅助分支仍需要针对特定任务的标注数据进行监督训练。
