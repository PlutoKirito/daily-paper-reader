## 摘要
无损模型压缩在缓解位精确（bit-exact）大语言模型（LLM）服务中的内存和带宽瓶颈方面具有巨大潜力。然而，由于与 GPU 架构存在根本性的设计不匹配，现有方法通常会导致显著的推理延迟：在算子层面，传统熵编码器产生的变长比特流破坏了 SIMT 并行性；在系统层面，解耦的流水线导致了冗余的内存流量。我们提出了 ZipServ，这是一个为高效 LLM 推理协同设计的无损压缩框架。ZipServ 引入了张量核心感知的三重位图编码（TCA-TBE），这是一种新型的定长格式，可实现常数时间内的并行解码，并配合融合解压-GEMM（ZipGEMM）算子，将权重实时解压并直接存入 Tensor Core 寄存器。这种“加载压缩数据，计算解压数据”的设计消除了中间缓冲区，并最大化了计算强度。实验表明，ZipServ 可将模型大小减少高达 30%，在算子层面相比 NVIDIA 的 cuBLAS 实现高达 2.21 倍的加速，并在端到端推理上相比 vLLM 平均提速 1.22 倍。ZipServ 是首个在 GPU 上为 LLM 推理同时提供存储节省和显著加速的无损压缩系统。

## Abstract
Lossless model compression holds tremendous promise for alleviating the memory and bandwidth bottlenecks in bit-exact Large Language Model (LLM) serving. However, existing approaches often result in substantial inference slowdowns due to fundamental design mismatches with GPU architectures: at the kernel level, variable-length bitstreams produced by traditional entropy codecs break SIMT parallelism; at the system level, decoupled pipelines lead to redundant memory traffic. We present ZipServ, a lossless compression framework co-designed for efficient LLM inference. ZipServ introduces Tensor-Core-Aware Triple Bitmap Encoding (TCA-TBE), a novel fixed-length format that enables constant-time, parallel decoding, together with a fused decompression-GEMM (ZipGEMM) kernel that decompresses weights on-the-fly directly into Tensor Core registers. This "load-compressed, compute-decompressed" design eliminates intermediate buffers and maximizes compute intensity. Experiments show that ZipServ reduces the model size by up to 30%, achieves up to 2.21x kernel-level speedup over NVIDIA's cuBLAS, and expedites end-to-end inference by an average of 1.22x over vLLM. ZipServ is the first lossless compression system that provides both storage savings and substantial acceleration for LLM inference on GPUs.

*   **核心问题**：大语言模型（LLM）推理主要受限于 GPU 的**内存容量**和**内存带宽**。
*   **研究背景**：
    *   **有损压缩的缺陷**：量化（Quantization）和剪枝（Pruning）虽能减小模型，但会造成精度损失，在安全敏感或长文本推理场景中表现不佳。
    *   **无损压缩的瓶颈**：现有的无损压缩（如 Huffman 或 ANS 编码）在 GPU 上表现极差。原因在于：1) **算子层级不匹配**：变长比特流导致 SIMT 架构下的线程分支分叉（Divergence），破坏并行性；2) **系统层级不匹配**：采用“先解压到显存、再读取计算”的解耦流水线，产生了冗余的内存流量，抵消了压缩带来的带宽节省。
*   **整体含义**：本文旨在通过硬件感知的协同设计，使无损压缩不仅能节省存储，还能直接加速 GPU 推理。

### 2. 方法论：核心思想与关键技术
ZipServ 的核心思想是利用 LLM 权重中 BF16 格式指数位的统计冗余，设计一种对硬件友好的定长编码格式。
*   **TCA-TBE（张量核心感知三重位图编码）**：
    *   **统计发现**：BF16 权重的指数位分布极度倾斜，Top-7 的指数值覆盖了超过 95% 的权重，且这些指数值在数值上通常是连续的。
    *   **定长编码**：放弃变长编码，为每个元素分配固定的 3-bit 编码。
    *   **三重位图布局**：将 3-bit 编码拆分为三个独立的 64-bit 位图（Bitmaps），每个位图代表一个位平面。这种设计确保了内存访问对齐，且解码时无需分支判断。
*   **ZipGEMM（融合解压-GEMM 算子）**：
    *   **寄存器级解压**：权重以压缩格式从显存加载，在 GPU 寄存器中直接进行实时（On-the-fly）解压，随后立即喂给 Tensor Core 的 `mma.sync` 指令。
    *   **计算强度提升**：通过消除中间显存缓冲区，显著提高了 Roofline 模型中的计算强度（Compute Intensity）。
*   **阶段感知推理策略**：
    *   **Decode 阶段**：使用 ZipGEMM 融合算子，解决带宽瓶颈。
    *   **Prefill 阶段**：由于是计算密集型，采用高效的独立解压算子将权重还原至显存，再调用标准 cuBLAS，以分摊解压开销。

### 6. 主要结论与发现
*   **存储节省**：ZipServ 可将模型权重体积减少约 **25% - 30%**（例如 LLaMA-3.1-70B 从 131GB 降至 93GB）。

### 7. 优点：亮点与创新
*   **软硬协同设计**：深刻理解了 Tensor Core 的寄存器布局，将解压逻辑与矩阵乘法指令深度融合，打破了无损压缩“慢”的固有印象。
*   **定长编码创新**：TCA-TBE 巧妙地利用位平面（Bit-plane）设计，规避了 GPU 处理变长数据时的线程分叉问题。
*   **位精确保证**：与量化不同，ZipServ 保证了模型输出与原始 BF16 模型完全一致，无任何精度损失。

### 8. 不足与局限
*   **计算密集型场景收益有限**：在 Prefill 阶段或 Batch Size 极大（计算密集）时，解压带来的 ALU 开销可能抵消带宽节省的收益。
*   **高带宽硬件优势减弱**：在配备 HBM3 等极高带宽的数据中心 GPU（如 H100）上，由于内存瓶颈不明显，ZipServ 的加速效果不如在消费级显卡上突出。
*   **算子覆盖**：目前主要针对线性层（GEMM）进行了深度优化，对于 Attention 算子或其他非线性层的压缩支持尚在讨论阶段。
