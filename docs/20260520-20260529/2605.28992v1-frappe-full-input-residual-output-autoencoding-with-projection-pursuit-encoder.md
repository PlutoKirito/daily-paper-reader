## 摘要
媒体压缩标准在率失真-复杂度权衡方面已达到瓶颈，限制了在机器人、可穿戴设备和遥感等应用中将昂贵的AI感知卸载到云端的可能性。基于DNN的编解码器提高了压缩效率，但代价是：它们难以轻松适应可用比特率的较大变化，并且实时编码需要昂贵且耗电的GPU，这阻碍了在低成本或资源受限平台上的使用。为了解决这些限制，我们提出了一种新颖的自编码框架（FRAPPE），该框架使用全输入通过投影追踪编码器预测残差输出。FRAPPE的编码目标自然地按重要性对潜在通道进行排序，允许零开销的可变速率编码。与基于RNN的学习型编解码器（其编码器消耗前一个重建的残差）或RVQ风格编解码器（其码本必须顺序应用）不同，FRAPPE的分析路径是一个由独立输入投影组成的极易并行化的有向无环图。使用FRAPPE，我们构建了一个可变速率RGB图像编解码器（FRAPPE-Image），并评估其与标准图像编解码器相比的率失真-复杂度权衡。在高压缩比（约0.1 bpp）下，FRAPPE-Image提供了比AVIF更高的感知质量，编码速度加快47倍，使其能够在仅CPU上实现实时1080p、30fps编码。我们的代码和预训练模型可在以下网址获取：https://github.com/UT-SysML/FRAPPE。

## Abstract
Media compression standards have reached a plateau in terms of the rate-distortion-complexity trade-off, limiting the ability to offload expensive AI perception to the cloud in applications like robotics, wearables, and remote sensing. DNN-based codecs improve compression efficiency, but at a cost: they cannot easily adapt to large changes in available bitrate, and real-time encoding requires expensive, power-hungry GPUs that prohibit use on low-cost or resource-constrained platforms. To address these limitations, we propose a novel autoencoding framework (FRAPPE) that uses the Full input to predict the Residual output via a Projection Pursuit Encoder. FRAPPE's encoding objective naturally sorts latent channels by importance, allowing zero-overhead variable-rate coding. Unlike RNN-based learned codecs, whose encoder consumes the previous reconstruction's residual, or RVQ-style codecs, whose codebooks must be applied sequentially, FRAPPE's analysis path is an embarrassingly parallel DAG of independent input projections. Using FRAPPE, we build a variable-rate RGB image codec (FRAPPE-Image), and evaluate its rate-distortion-complexity trade-off against standard image codecs. At high compression ratios (approx. 0.1 bpp) FRAPPE-Image provides higher perceptual quality than AVIF with 47 times faster encoding, making it capable of real-time 1080p, 30fps CPU-only encoding. Our code and pre-trained models are available: https://github.com/UT-SysML/FRAPPE .

## 2. 论文提出的方法论

### 核心思想
FRAPPE（Full Input, Residual Output Autoencoding with Projection-Pursuit Encoder）是一种新型残差自编码框架，借鉴**投影追踪回归（PPR）**思想：利用**完整输入**预测**残差输出**，并通过逐通道拟合输出空间残差，自动实现潜在通道的重要性排序，从而获得零开销变速率能力。

### 关键技术细节

#### 代码工作流
编码过程由分析变换 \(G_A\)、压缩非线性 \(\Phi\)、量化器 \(Q\) 和合成变换 \(G_S\) 组成：
\[
\hat{x} = G_S \circ \text{Adapt}_{p_d} \circ Q \circ \Phi \circ G_A(x)
\]

- **分析变换** \(G_A\)：将输入 \(x\) 分解为 \(S\) 个尺度组，每组 \(n_s\) 个潜在通道，每个通道对应一个学习到的线性投影（非重叠块）。所有投影可合并为单步跨步卷积，形成**无递归的并行 DAG**。
- **压缩非线性** \(\Phi\)：软符号压缩函数，将激活值映射到有符号 8-bit 范围（\(r=127\)），便于量化。
- **量化** \(Q\)：训练时用加性噪声模拟，推理时直接取整。
- **合成变换** \(G_S\)：重型非线性解码器（ConvNeXt 残差块 + 点卷积 + 上采样），吸收绝大部分参数和 FLOPs。
- **熵编码**：每个尺度组独立编码，使用 JPEG-LS（近似拉普拉斯分布），CPU 友好。

#### 逐通道残差拟合（训练流程）
- 定义合并编解码器 \(F_{m-1}\)（前 \(m-1\) 通道），初始 \(F_0 \equiv 0\)。
- 第 \(m\) 通道的编码器-解码器对拟合输出空间残差 \(\ r_m = x - F_{m-1}(x)\)，最小化：
  \[
  \mathcal{L}_m = \log_{10} \| r_m - \hat{r}_m \|_2^2 + \lambda_m \mathbb{E}[r_m^2]^\rho \log_2 \text{Std}(\Phi(\omega_m^\top \text{Patch}_{p_m}(x)))
  \]
  其中 \(\hat{r}_m\) 为单通道预测，第二项是 LiVeAction 的 log-variance 率代理，并用残差功率 \(\mathbb{E}[r_m^2]^\rho\) 重新加权（\(\rho=0.3\)），防止后期通道率项主导。
- 按粗到细顺序训练：早期通道 patch 大（如 32×32）、 \(\lambda_m\) 大（压缩比高）；后期通道 patch 小（如 2×2）、 \(\lambda_m\) 小。
- **自然排序**：每个新通道只需捕捉未被前序通道解释的残差方差，从而通道按重要性自动排序。

#### 零开销可变速率
- 训练完成后，保留前 \(n\) 通道并选择对应的**快照解码器**（每个 \(n\) 一个），即可得到不同率点，无需重新编码或微调解码器。
- 编码器权重对所有 \(n\) 是位相同的。

#### 低成本编码器分析
- 每像素成本：\(CN + \sum_s 4n_s/p_s^D\)，对 FRAPPE-Image 约为 **68 ops/pixel**。
- 所有尺度组的卷积共享输入 \(x\) 且相互独立，可并行或流水线化。

## 6. 主要结论与发现

1. **极端压缩比**：FRAPPE-Image 可实现 5000:1 以上的压缩比（0.008 bpp），AVIF/JPEG 最低仅 352:1/139:1。
2. **感知质量领先**：在 ~0.1 bpp 时，FRAPPE-Image 的 DISTS 优于 AVIF，编码速度快 47 倍（915 MPx/s vs 6.04 MPx/s）。
3. **实时 CPU 编码**：在 n=20（约 31.5 dB PSNR）时，吞吐量 73.6 MPx/s，足以支持 1080p 30fps 纯 CPU 编码。相比之下，DCVC-RT 需高功率 GPU。
4. **编码效率对比**：
   - 在低比特率（<0.1 bpp）区域，FRAPPE 在所有感知指标上最强。
   - 在中高比特率（0.1–1 bpp），mbt2018 的 PSNR/SSIM 领先（+2.2~4.2 dB），但速度慢 ~1000 倍。
   - AVIF 在匹配比特率时 PSNR 略优，但 FRAPPE 在感知阈值更低的应用中优势更大。

## 7. 优点

- **方法论创新**：将投影追踪回归引入残差自编码，自然排序通道重要性，零开销变速率。
- **并行编码路径**：分析变换由独立投影组成，可合并为单次跨步卷积，无递归/序列化，极大降低编码延迟。
- **不对称部署**：编码器极轻量（68 ops/pixel），解码器重型，完美适配传感器-云端场景。
- **硬件友好**：不需要 GPU；8-bit 量化和简单熵编码适合低功耗 MCU。
- **分析透彻**：提供了详细的 BD 分析表格和不同比特率/质量区间对比，论证清晰。

## 8. 不足与局限

- **应用范围有限**：仅实验验证 RGB 图像；音频、视频、高光谱、3D 体积等未涉及。
- **高比特率劣势**：在中等及高比特率（>0.3 bpp）时，PSNR/SSIM 明显低于 mbt2018 和 AVIF，不适合追求高保真度的场景。
- **存储开销**：每个通道数 n 需要一个解码器快照（FRAPPE-Image 有 21 个），部署时存储压力大；未使用随机通道丢弃训练（如 MCUCoder）来解决。
- **熵编码优化空间**：JPEG-LS 简单但非最优；替换为学习型熵模型可能提升率失真性能。
- **训练资源未透明**：缺乏 GPU 型号、数量及训练时长的公开信息，影响可复现性。
- **公平性细节**：mbt2018 使用似然 bpp（上限），AVIF 使用默认设置，可能不完全公平。作者已说明，但读者需注意。
- **消融实验缺失**：未系统评估不同组件（如 compander、ρ、λ 调度）的贡献。
